4. CSS & LESS

By default, SMART is loading a complete stylesheet file that contains all the styles needed for the entire template, which is located within the less/module_name folder.

For example, the assets/less/admin/module.admin.stylesheet-complete.less used by the admin module, might contain something like this:

// Example of stylesheet complete contents
@import '../../../assets/components/menus/navbar.main.less';
@import '../../../assets/components/menus/skin.colors.less';
@import '../../../assets/components/ui_breadcrumb/breadcrumb.less';
@import '../../../assets/components/ui_forms/forms.less';
@import '../../../assets/components/ui_labels/labels.less';
...

The imports represent all the style assets used by over 300 components included with the template.


5. Assets Structure

The assets is structured in a specific modular way.

smart/

└── assets/
    │   // Library includes core frameworks like Bootstrap, jQuery, jQuery UI, Modernizr etc.
    └── library/
    │
    │   // Plugins like jQuery Plugins, Bootstrap Plugins, etc
    └── plugins/
    │
    │   // Custom Javascript & Styling
    └── components/
        │
        │   // As a general rule of thumb, component folders are named by the following
        │   // convention: {module_or_page_name}_{component_categories}_{component_name}
        └── component_name/
                    └── <custom javascript>
                    └── <custom style>
